# Декораторы логирования — Учебный проект Нетологии

Этот проект создан в рамках учебного курса по Python в [Нетологии](https://netology.ru/).  
Цель — реализация двух декораторов для логирования вызовов функций:

- `logger_1` — записывает логи в файл `main.log` без передачи аргументов.
- `logger_2(path)` — записывает логи в указанный файл.

## Функционал

- Логирование:
  - Имени функции
  - Позиционных и именованных аргументов
  - Результата выполнения
  - Времени вызова (в формате `ГГГГ-ММ-ДД ЧЧ:ММ:СС`)
- Обработка ошибок записи в файл
- Человеко-читаемый формат логов

## Тесты

В проекте реализованы автоматические тесты:
- `test_1()` — проверяет `logger_1`
- `test_2()` — проверяет `logger_2` на нескольких файлах

Тесты проверяют:
- Создание файла лога
- Корректность записи данных
- Соответствие формата

## Пояснение по логам

При использовании `@logger_1` с рекурсивным генератором `flat_recursive_generator` 
в лог попадают **все вызовы функции**, включая рекурсивные. 
Поэтому в `main.log` может быть много записей (например, 30+), что **не является ошибкой**, 
а отражает глубину вложенности и количество вызовов.

## Дополнительный модуль

В проекте присутствует отдельный модуль `using_a_decorator.py`, демонстрирующий применение декоратора `@logger_1` к реальным функциям — в частности, к генераторам списков с различной вложенностью.  
Этот модуль:
- Импортирует `logger_1` из `log_decorators`
- Применяет его к функциям `flat_generator` и `flat_recursive_generator`
- Содержит тесты `test_3()` и `test_4()`, подтверждающие корректность работы
